# Установка окружения и запуск ноутбуков (uv)

В курсе мы используем `uv` как основной инструмент для управления Python-окружением и зависимостями.

## 0) Предпосылки

Нужно:

- Python 3.11+ (если у вас уже есть Python – ок)
- `uv` установлен в системе

Проверка:

```bash
python --version
uv --version
```

Если `uv` не установлен – установите по официальной инструкции uv (под вашу ОС).

---

## 1) Быстрый старт (рекомендуемый сценарий)

### 1.1. Клонируйте репозиторий курса

```bash
git clone <URL_ЭТОГО_РЕПОЗИТОРИЯ>
cd mlsec-course-meta
```

### 1.2. Создайте локальное окружение проекта

В корне репозитория выполните:

```bash
uv venv
```

Активируйте окружение:

- **Windows (PowerShell):**

```powershell
. .\.venv\Scripts\Activate.ps1
```

- **Windows (cmd):**

```bat
.\.venv\Scripts\activate.bat
```

- **macOS / Linux (bash/zsh):**

```bash
source .venv/bin/activate
```

Проверьте, что Python берётся из `.venv`:

```bash
python -c "import sys; print(sys.executable)"
```

---

## 2) Установка базовых пакетов для семинаров

Минимальный набор для большинства семинаров:

```bash
uv pip install -U pip
uv pip install jupyter ipykernel numpy pandas matplotlib scikit-learn
```

Если в конкретном семинаре нужны дополнительные пакеты – они будут указаны в `seminars/Sxx/Sxx-brief.md`.

---

## 3) Запуск Jupyter

### Вариант A (обычный): Jupyter Notebook

```bash
jupyter notebook
```

### Вариант B: JupyterLab

Если вам удобнее JupyterLab:

```bash
uv pip install jupyterlab
jupyter lab
```

---

## 4) Открытие ноутбуков семинара

Например, для семинара S03:

- Откройте `seminars/S03/S03-demo.ipynb`
- Запустите все ячейки по порядку

---

## 5) Правила воспроизводимости (важно)

- Если в ноутбуке есть `RANDOM_STATE` / `seed` – не меняйте его без причины.
- Если меняете параметры/разбиение – фиксируйте это в выводах/заметках.
- Не подгоняйте решение под тест/проверку: цель – корректный протокол и честные результаты.

---

## 6) Если вы не хотите ставить всё на систему

Допускается запуск в облаке (например, Colab).
Но приоритет – локальный запуск через `uv`, чтобы окружение было одинаковым.
