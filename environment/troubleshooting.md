# Troubleshooting: типовые проблемы (uv и Jupyter)

Ниже – самые частые ошибки и быстрые способы их диагностировать.

---

## 1) `uv: command not found` / `uv` не запускается

**Причина:** `uv` не установлен или не в PATH.

**Что делать:**

- Установите `uv` по официальной инструкции для вашей ОС.
- Перезапустите терминал/IDE после установки.
- Проверьте:

```bash
uv --version
```

---

## 2) Активировал окружение, но пакеты "не видны"

**Симптом:** `import numpy` / `import pandas` падает, хотя вы "ставили".

**Причина:** пакеты поставлены не в то окружение или окружение не активировано.

**Проверка:**

```bash
python -c "import sys; print(sys.executable)"
python -m pip --version
```

Путь должен указывать на `.venv`.

**Решение:**

1. Активируйте `.venv` (см. `setup.md`).
2. Ставьте пакеты через:

```bash
uv pip install numpy pandas matplotlib scikit-learn
```

---

## 3) Jupyter запускается, но Kernel не из `.venv`

**Симптом:** ноутбук открывается, но импортов нет / версии не те.

**Причина:** Jupyter видит другой Python.

**Решение (рекомендуется):**
Установите ipykernel в активированном окружении и зарегистрируйте kernel:

```bash
uv pip install ipykernel
python -m ipykernel install --user --name mlsec-uv --display-name "Python (mlsec-uv)"
```

Дальше в ноутбуке: **Kernel – Change Kernel – Python (mlsec-uv)**.

---

## 4) `ModuleNotFoundError: No module named 'jupyter'`

**Решение:**

```bash
uv pip install jupyter
jupyter notebook
```

---

## 5) Ошибки установки пакетов (Windows): "Microsoft Visual C++ Build Tools"

**Симптом:** падает сборка колёс (wheels), особенно на некоторых пакетах.

**Что делать:**

- Сначала обновите pip и попробуйте снова:

```bash
uv pip install -U pip
```

- По возможности ставьте популярные пакеты, у которых есть prebuilt wheels.
- Если конкретный пакет требует компиляции – напишите в чат курса/преподавателю (дадим обходной вариант или зафиксируем версии).

---

## 6) Конфликт версий / "works on my machine"

**Симптом:** у вас всё работает, у другого студента – нет.

**Диагностика:**
Сохраните версии:

```bash
python --version
uv --version
python -c "import numpy, pandas, sklearn; print(numpy.__version__, pandas.__version__, sklearn.__version__)"
```

**Решение:**

- Пересоздайте окружение "с нуля":

  1. удалите папку `.venv`
  2. `uv venv`
  3. поставьте пакеты заново

---

## 7) `PermissionError` / политика PowerShell не даёт активировать venv (Windows)

**Симптом:** PowerShell ругается на выполнение скриптов при `Activate.ps1`.

**Решения:**

- Запустите PowerShell от имени пользователя и разрешите выполнение для текущей сессии:

```powershell
Set-ExecutionPolicy -Scope Process -ExecutionPolicy Bypass
```

- Или используйте `cmd` и `activate.bat`.

---

## 8) "Файл ноутбука не открывается" / "NotJSONError"

**Причина:** файл повреждён или скачан некорректно.

**Что делать:**

- Перекачайте файл / сделайте `git pull`.
- Не редактируйте `.ipynb` в редакторе, который портит JSON.

---

## 9) Медленно выполняется / зависает ноутбук

**Причины:**

- очень большой датасет,
- слишком тяжёлая модель,
- бесконечный цикл/слишком много итераций.

**Что делать:**

- остановить kernel и перезапустить (Kernel – Restart).
- уменьшить размер данных/число итераций (если это не ломает смысл задания).
- проверьте, что не запускаете обучение "внутри цикла" без необходимости.

---

## 10) Куда писать, если не помогло

Чтобы вам быстро помогли, прикладывайте:

- ОС (Windows/macOS/Linux),
- вывод:

```bash
python --version
uv --version
python -c "import sys; print(sys.executable)"
```

- полный traceback ошибки (скрин или текст),
- что именно вы пытались сделать (команды).
